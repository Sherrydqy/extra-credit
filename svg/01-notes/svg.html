SVG

Vector Graphics

  • defined by points, lines, curves, shapes
  • can be scaled up with no loss of quality
  • ideal for logos, graphics, charts
  • edited using Illustrator

Scalable Vector Graphics

  • vector graphics described in XML
  • can be integrated in HTML
  • well supported in modern browsers

eXtensible Markup Language (XML)

  • rules for creating other markup languages (like HTML, SVG, etc.)
  • follows a strict syntax
    • element, attributes lowercase
    • elements must close
    • attributes must be double quoted
    • proper nesting is enforced
    • special characters must be encoded

Using SVG on the Web

  • add using <img> tag
  • embed using <svg> tag
  • also recognized in css

Styling SVG

  • svg attributes
  • inline styles
  • using <style>
  • external style sheets

SVG Properties

SVG is its own markup language so it has attributes (and values) unique to itself. When styling, you may be inclined to use HTML properties. Don't. They probably will not work.

Use fill instead of color; stroke instead of border.

See: SVG Attribute Reference

svg

The svg element define where our vector graphics will be rendered.

It is necessary to define its dimensions.

<svg width="600" height="300"></svg>

svg coordinates

  • all drawing begins in the top, left corner
  • x-axis runs horizontally
  • y-axis runs vertically

rect

  • draws a rectangle
  • x, y: the starting point (top, left)
  • width, height: dimensions

<rect x="20" y="10" width="100" height="50" />

circle

  • draws a circle/ellipse
  • cx, cy the starting point from the center
  • r: radius
  • rx, ry: radius for x-axis, y-axis (ellipse only)

<circle cx="200" cy="100" r="20" />

line

  • draws a line
  • x1, y1: starting point
  • x2, y2: end point
  • stroke: required to make line visible

<line x1="20" y1="20" x2="50" y2="50" stroke="black" />

polyline & polygon

A polyline is a group of connected, straight lines. A polygon is similar, except the path returns to the first point, creating a closed shape.

<polyline points="60 110, 65 120, 70 115 ..." />

<polygon points="50 160, 55 180, 70 180, 60 190 ..."/>

path

  • draws crooked, curved lines
  • d provides points along the path
    • M: move to
    • L: line to
    • Z: close path
    • C: curve
    • A: arc

<path d="M 13 100 L 60 50 L 90 90 L 140 30" />

defs, symbol and use

defs defines reusable graphic objects (paths, shapes, gradients, etc.) for re-use within an SVG.

symbol defines reusable graphic objects for re-use throughout the document.

In both cases, use is what calls that definition.

symbol Example

<svg>
  <symbol id="box" viewBox="0 0 50 50">
    <rect x="0" y="0" width="50" height="50" />
  </symbol>
</svg>

<svg>
  <use xlink:href="#box" />
</svg>

defs Example

<svg viewBox="0 0 50 50">>
  <defs>
    <circle cx="0" cy="0" r="5" id="circ" />
  </defs>

  <use x="10" y="10" xlink:href="#circ" />
</svg>

g

The g (grouping) element groups similar or related svg elements.

It is useful for structuring, applying styles to child elements and transforms (i.e. rotate, skew, etc.)

<svg width="100" height="100">
  <g>
    <circle cx="50" cy="50" r="25" fill="red" />
    <circle cx="50" cy="50" r="20" fill="white"/>
    <circle cx="50" cy="50" r="15" fill="red" />
    <circle cx="50" cy="50" r="10" fill="white"/>
    <circle cx="50" cy="50" r="5" fill="red" />
  </g>
</svg>

text

The text element draws a graphic element which contains text.

<text x="5" y="12">Hello</text>
